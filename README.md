# Проектная работа: Кейс «Крутые уборщики»

**Студент:** Рыженков  
**Курс:** Инженер баз данных  
**Тема:** Проектирование хранилища данных, ETL и дашбордов для прогнозирования кассовых разрывов

## Описание кейса

Компания «Крутые уборщики» предоставляет услуги выездной мойки автомобилей в Москве и Санкт-Петербурге.  
**Проблемы as-is:**
- Ручное сведение отчётности в Excel, главбух тратит много времени;
- Нет автоматического прогноза выручки - кассовые разрывы;
- Данные разрознены: две CRM (Мск, СПб) и 1С, интеграция отсутствует.

**Цель to-be:**
- Создать единое хранилище данных по модели "снежинка";
- Автоматизировать ETL-процесс - ежедневная загрузка из источников;
- Построить дашборд в Yandex DataLens с прогнозом выручки и отслеживанием кассовых разрывов;
- Прогнозная модель – "чёрный ящик" (внешняя).

# Технологический стек

- **СУБД:** PostgreSQL (учебный сервер vpngw.avalon.ru)
- **ETL:** Python (pandas, psycopg2) + Apache Airflow
- **BI:** Yandex DataLens
- **Моделирование:** Draw.io
- **Контроль версий:** GitHub

## Архитектура хранилища (DWH)

Модель – **снежинка** (snowflake).  
Схема: `dwh` в базе данных `DE2026_Ryzhenkov`.

### Измерения
- `dim_city` – города (56 – Москва, 324 – Санкт-Петербург)
- `dim_date` – календарь (2021–2024)
- `dim_client` – клиенты
- `dim_service` – услуги
- `dim_seller` – продавцы/менеджеры
- `dim_currency` – валюты
- `dim_status` – статусы заказов (0 – Неизвестен, 1 – Открыт, 2 – Выполнен, 3 – Оплачен)
- `dim_expense_category` – категории расходов

### Факты
- `fact_sales` – продажи (сделки из CRM Москва и СПб)
- `fact_expenses` – расходы из 1С
- `fact_income` – поступления из 1С

### Прогнозный слой
- `forecast_sales` – прогноз выручки 

## Инструкция по развёртыванию

### 1. Создание базы данных 

Подключиться к серверу и выполнить:

CREATE DATABASE "DE2026_Ryzhenkov" ENCODING 'UTF8';

### 2. Создание таблиц
В DBeaver (или другом клиенте) выполнить скрипт sql/01_create_table_de2026.sql в базе DE2026_Ryzhenkov.

### 3. Установка зависимостей Python
pip install pandas psycopg2-binary

### 4. Настройка подключений в etl_script.py
Отредактировать файл etl/etl_script.py – заменить '****' на реальный пароль от базы данных во всех секциях.

### 5. Запуск ETL 
python etl/etl_script.py
Процесс может занять какое-то время, зависит от объёма данных. После успешного завершения проверить таблицы в dwh.

## Автоматизация с Apache Airflow

### 1. Разместить DAG
Скопировать файл dags/project_work_etl_dag.py в папку dags Airflow (например, C:\Users\Ivan\airflow_docker\dags\).

### 2. Настройка расписания
DAG настроен на ежедневный запуск в 02:00 (schedule_interval='0 2 * * *'). При желании изменить параметр.

### 3. Запуск через Airflow
- Открыть веб-интерфейс Airflow (обычно http://localhost:8080).
- Найти DAG project_work_etl.
- Запустить.
- Для ручного теста нажать Trigger DAG.
- Логи выполнения можно посмотреть в интерфейсе Airflow.

## Результаты и выводы

**- Разработано единое хранилище данных, объединяющее две CRM и 1С.**
